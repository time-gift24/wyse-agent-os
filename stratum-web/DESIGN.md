# 设计系统：运筹 Stratum

## 1. 视觉定位

运筹是一套克制、可信、紧凑的 Agent 产品工作台。信息密度为 Daily App Balanced（6/10），布局差异度为 Balanced Product（5/10），动效强度为 Focused Feedback（3/10）。最终视觉结果参考本地 Tavily 复刻站点：固定侧栏、顶部状态区、受控内容画布、暖色中性层级、单一产品蓝和细边界共同建立产品连续性。

Lovable 继续承担暖中性色、基础排版与克制纵深。Tavily 参考决定最终布局比例、密度、空间、图标与交互观感。Stratum 既有 Cloud Dancer、Baltic Sea、Blue Fusion、Quiet Violet 等多色值只保留给品牌标识和未来有明确沟通目的的特效，不进入普通产品表面。

视觉参考不是代码依赖。不得复制 Tavily 品牌、Logo、业务文案、图片、额度、用量、模拟数据或缺少授权证明的字体文件。

## 2. 配色与语义职责

颜色分为产品色、语义状态色和保留特效色。每种颜色只承担一种职责。

### 浅色主题

- **Canvas** `#fefcf5`：页面主画布。
- **Sidebar** `#fdf8ec`：固定侧栏与导航抽屉。
- **Surface** `#fffcf6`：顶部状态区、Composer 与主要浮层。
- **Surface Subtle** `#f4f0e6`：选中项、加载骨架和轻强调区域。
- **Ink** `#1f1e1e`：主文本。
- **Ink Secondary** `#625f59`：辅助文本，使用前必须验证 4.5:1 对比度。
- **Border** `rgba(31, 30, 30, 0.10)`：普通边界。
- **Border Strong** `rgba(31, 30, 30, 0.20)`：输入、焦点外层与重要分隔。
- **Primary Blue** `#2677ff`：品牌蓝、当前选择和焦点环。
- **Primary Action** `#1e65d9`：白字填充主要操作，对比度约 5.36:1。
- **Primary Action Hover** `#1958c1`：填充主要操作悬停。
- **Primary Foreground** `#ffffff`：填充主要操作文字。

### 暗色主题

- **Canvas** `#161515`：页面主画布。
- **Sidebar** `#1f1e1e`：固定侧栏与导航抽屉。
- **Surface** `#242222`：顶部状态区、Composer 与主要浮层。
- **Surface Subtle** `#2b2929`：选中项、加载骨架和轻强调区域。
- **Ink** `#fffcf6`：主文本。
- **Ink Secondary** `#c0b9ae`：辅助文本。
- **Border** `rgba(255, 252, 246, 0.12)`：普通边界。
- **Border Strong** `rgba(255, 252, 246, 0.22)`：输入与重要分隔。
- **Primary Blue** `#4d8dff`：暗色主题主要操作、当前选择和焦点环。
- **Primary Blue Hover** `#73a5ff`：主要操作悬停。

### 语义状态色

- 绿色只表示连接就绪、执行成功或明确可用状态。
- 黄色只表示等待、警告或需要用户注意。
- 红色只表示错误、取消失败或破坏性操作。
- 蓝色信息状态与主要操作必须通过文案和图标区分，不能只依赖颜色。
- 无法判断状态时使用中性文字，不显示绿色成功点。

### 保留特效色

Stratum 多色 token 可以继续用于品牌标识。普通背景、导航、顶部状态区、Composer、会话列表、卡片、按钮和正文不得使用这些 token。本阶段不新增环境式特效。

## 3. 排版

- **展示与中文：** `Noto Sans Variable`。
- **正文与界面：** `Nunito Sans Variable`。
- **可靠回退：** `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。
- 未确认授权前不得从参考项目复制 Suisse Intl 字体文件。
- 正文行长保持在 65-75ch，对话正文最大阅读宽度约 800px。
- 标题使用 `text-wrap: balance`，长正文使用 `text-wrap: pretty`。
- 常规字重使用 400，强调与标题使用 600，不通过 300 或 700 制造额外层级。

| 角色 | 字号 | 字重 | 行高 | 字距 |
| --- | --- | --- | --- | --- |
| 工作台主标题 | 桌面 44px / 平板 36px / 移动 32px | 600 | 1.08 | -0.03em |
| 页面标题 | 32px | 600 | 1.15 | -0.02em |
| 区块标题 | 20px | 600 | 1.25 | -0.01em |
| 卡片或列表标题 | 16px | 600 | 1.4 | 0 |
| 正文与对话 | 16px | 400 | 1.55 | 0 |
| 导航与标准操作 | 14px | 400 或 600 | 1.45 | 0 |
| 说明文字 | 13px | 400 | 1.45 | 0 |

中英文使用相同语义层级。桌面按钮文案不得换行；空间不足时先缩短文案、增加可用宽度或按既定优先级截断。

## 4. 形状、边界与纵深

- 紧凑控件：8px 圆角。
- 导航项、按钮与小型输入：9px 圆角。
- 标准内容区块与 Composer：12px 圆角。
- 固定侧栏：14px 圆角。
- 顶部状态区与大型浮层：16px 圆角。
- 全圆角只用于头像、状态点、开关和圆形图标操作。
- 普通内容优先使用留白或单一分隔线，不为每组文字添加卡片。
- 标准容器使用低对比 1px 边界，不叠加大于 8px 模糊的装饰阴影。
- 固定侧栏可以使用极轻、与暖色背景同色相的 2px-8px 阴影。
- 浮层只在需要脱离内容流时使用克制纵深。
- 禁止卡片套卡片、32px 以上容器圆角、侧边粗色条、渐变文字和玻璃拟态。

## 5. 共享产品壳层

### 桌面结构

- 固定侧栏宽度约 285px，距视口顶部、左侧与底部 16px。
- 顶部状态区高度约 75px，位于主内容区域上方，最大可用宽度随侧栏偏移。
- 主内容画布最大宽度约 1024px，在侧栏之外的可用区域居中。
- 桌面断点建议为 `xl`（1280px）；在无法同时容纳侧栏、内容安全边距和对话阅读宽度时提前切换为移动结构。
- 层级顺序为：基础内容、固定壳层、抽屉遮罩、抽屉、通知与工具提示。禁止使用随意的高位 z-index。

### 侧栏内容

1. 品牌“运筹”。
2. 一级导航“概览、隆中对”。
3. 有真实数据时显示有限条最近会话，无数据时显示简洁空状态。
4. 底部放置语言、主题和注册或账户操作。

当前项使用轻中性选中表面、加深文字与对应 Lucide 图标。不要使用装饰状态点、粗侧线或与页面不一致的胶囊导航。

### 顶部状态区

- 左侧在移动端提供菜单按钮，桌面端直接显示当前页面标题或面包屑式上下文。
- 状态信息只能由真实模型、模板或连接结果推导。
- 已确认就绪时可以使用一个绿色语义点；加载、空、失败和未知必须各自可辨认。
- 右侧放置必要的语言、主题或账户操作，不重复侧栏中占用空间较大的内容。
- 不显示虚构额度、用量、版本号、天气、地点或装饰状态。

### 移动抽屉

- 固定侧栏退出布局，顶部状态区保留 44px 菜单触发器。
- 抽屉宽度不超过 285px 或视口宽度减 48px。
- 使用 Lucide `Menu` 与 `X`，不得使用文本字符模拟图标。
- 打开后锁定背景滚动，Escape、遮罩、关闭按钮与路由选择都可以关闭。
- 关闭后焦点返回菜单触发器；选择路由后焦点进入新页面主内容。
- `prefers-reduced-motion` 下立即完成开合。

## 6. 概览工作台

概览不再使用营销 Hero。桌面端采用不对称主次布局，移动端按任务优先级折叠为单列。

- 主区域：工作台标题、简短产品说明、进入隆中对的主要操作。
- 就绪区域：模型与 Agent 模板的真实加载、可用、空和错误状态。
- Agent 模板：展示后端真实返回内容，只提供应用当前支持的操作。
- 最近会话：展示本地持久化的真实标题、更新时间和恢复入口。
- 数据区块彼此独立，一个请求失败不阻塞其他内容和主要操作。
- 加载使用匹配最终布局的骨架，不使用页面中央旋转器。
- 空状态说明如何开始或配置，不生成示例数据。
- 禁止三张等宽功能卡片、营销指标、虚构进度条和卡片套卡片。

## 7. 隆中对工作台

- 隆中对始终位于共享产品壳层内。
- 对话主区域使用文档滚动，消息正文最大宽度约 800px。
- Composer 最大外宽约 896px，并相对主内容区域固定，而不是相对整个视口居中。
- 内容底部保留 Composer 高度与安全区，最后一条消息必须可以完整滚动到输入区上方。
- 桌面最近会话主入口位于侧栏，移动入口位于抽屉。现有历史数据和恢复逻辑继续复用。
- 助手消息直接落在画布上；用户消息只使用轻中性表面区分，不把整个对话包进大卡片。
- 思考、工具与中间步骤默认折叠，状态与审批事实始终可辨认。
- Agent、模型和状态控件在窄宽度下按优先级截断，发送与必要操作保持 44px 触控目标。
- 滚动到底部按钮只在用户离开真实底部时显示，使用共享 Button 与 Lucide `ArrowDown`。

## 8. 图标与控件

- 全产品只使用现有 `lucide-react` 图标家族。
- 默认图标视觉线宽约 1.8，常用尺寸为 16px、18px 和 20px。
- 菜单、关闭、下拉、发送、返回、主题、语言、会话和滚动操作不得使用文本字符或手写 SVG。
- 无可见文字的图标按钮必须提供可本地化的 `aria-label` 和 `title`（在 title 有帮助时）。
- 控件必须实现默认、悬停、焦点、按下、禁用、加载与错误状态。
- 主要操作使用产品蓝；次要操作使用透明或中性表面；破坏性操作只在真实破坏性行为中使用红色。
- 按下反馈使用约 1px 位移或 0.98 缩放，不动画布局尺寸。

受保护目录 `app/components/ui`、`app/components/react-bits` 和 `app/components/ai-elements` 的内部实现不得修改。通过语义 token、props、Stratum 自有包装层和使用方样式适配。

## 9. 动效与响应式

- 普通悬停、聚焦、选择、抽屉和状态反馈控制在 150-250ms。
- 简单状态使用 CSS；已有明确编排需求才使用 GSAP。
- 只在表达反馈、层级或状态迁移时添加动效，不做页面入场表演。
- 优先动画 `transform` 与 `opacity`，避免 `top`、`left`、`width` 和 `height`。
- 不监听 `window` scroll 持续更新 React state；使用现有滚动工具、IntersectionObserver 或库提供的机制。
- `prefers-reduced-motion: reduce` 下立即或近乎立即呈现最终状态。
- 小于桌面壳层断点时侧栏变抽屉；小于 768px 时所有多列内容明确收敛为单列。
- 页面使用 `min-height: 100dvh`，不得使用 `h-screen` 造成移动地址栏跳动。
- 页面、抽屉和 Composer 均不得产生横向滚动。

## 10. 可访问性与验证

- 正文与占位符对比度至少 4.5:1，大号文本至少 3:1。
- 键盘焦点必须在暖色与暗色表面上清晰可见。
- 所有图标按钮和菜单项具有可本地化名称。
- 触控目标至少 44px，桌面紧凑视觉可以通过外层命中区实现。
- 加载、空、错误、禁用、选中和成功状态不能只依赖颜色。
- 每次前端变更必须验证概览与隆中对、桌面与移动、浅色与暗色、中文与英文。
- 最终视觉检查重点包括 285px 侧栏、75px 顶部状态区、1024px 主画布、14/16/44px 排版、单一产品蓝、8-16px 圆角和页面间连续性。

## 11. 禁止项

- 不使用 emoji、Inter、通用衬线字体或纯黑。
- 不使用紫蓝霓虹、外发光、高饱和渐变、全局颗粒、玻璃效果、厚阴影或装饰网格。
- 不创建三等分通用卡片、Hero 指标、卡片套卡片、侧边粗色条或 32px 以上容器圆角。
- 不伪造用量、额度、模型、模板、会话、工具状态、审批说明或成功结果。
- 不复制 Tavily 品牌资产、文案、图片或未授权字体。
- 不把工具事件、审批详情或开发者调试列放入共享侧栏。
- 不直接修改受保护可复用组件内部实现。
