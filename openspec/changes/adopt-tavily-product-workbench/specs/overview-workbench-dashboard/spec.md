## ADDED Requirements

### Requirement: 真实产品概览
概览页 SHALL（必须）用现有模型、Agent 模板和本地最近会话数据构成真实产品工作台，而不是继续渲染临时营销 Hero。

#### Scenario: 概览加载成功
- **WHEN** 模型、Agent 模板与本地最近会话完成读取
- **THEN** 页面必须以明确主次层级展示产品入口、就绪信息、真实模板与最近会话，并提供进入隆中对的主要操作

#### Scenario: 无可展示数据
- **WHEN** 模型、模板或最近会话为空
- **THEN** 对应区块必须显示有行动意义的空状态或配置提示，不得使用静态示例、随机数字或虚构活动填充

### Requirement: 独立的数据状态
概览页 MUST（必须）让模型、模板和最近会话区块分别处理加载、成功、空和失败状态，避免单个数据源失败导致整个页面不可用。

#### Scenario: 部分请求失败
- **WHEN** 模型或模板中的一个请求失败，而其他数据仍可用
- **THEN** 失败区块必须提供真实错误说明或重试入口，其他可用区块与进入隆中对的主要操作必须继续工作

#### Scenario: 请求进行中
- **WHEN** 某个概览数据源仍在加载
- **THEN** 只有依赖该数据源的区域必须显示与最终布局相符的加载状态，不得把整个页面替换为全屏等待

### Requirement: 不对称工作台层级
概览页 SHALL（必须）使用接近 Tavily 参考的不对称主次布局，使主要入口明显强于辅助状态，同时避免通用 SaaS 卡片堆叠。

#### Scenario: 桌面布局
- **WHEN** 概览页在桌面视口渲染
- **THEN** 主要入口必须占据更大视觉面积，状态、模板或最近会话必须作为辅助内容组织，不得采用三张等宽卡片、卡片套卡片或营销 Hero 加功能卡片的结构

#### Scenario: 移动布局
- **WHEN** 概览页在移动视口渲染
- **THEN** 内容必须按任务优先级折叠为单列，主要操作必须保持可见，文字与控件不得出现横向裁切

### Requirement: 概览操作保持真实
概览页的所有操作 MUST（必须）对应现有路由、真实模板或可恢复会话。

#### Scenario: 启动隆中对
- **WHEN** 用户激活主要操作或选择可用 Agent 模板
- **THEN** 前端必须进入现有隆中对流程，并只传递当前应用真实支持的 Agent 或模板信息

#### Scenario: 选择最近会话
- **WHEN** 用户选择本地最近会话
- **THEN** 前端必须尝试恢复对应真实会话；若不可恢复，必须清理或标记失效引用并向用户提供明确反馈
