## 背景

Stratum Web 目前只有概览与隆中对两个路由。概览页仍是临时 Hero，隆中对使用居中聊天布局；上一轮 `unify-stratum-visual-language` 已经统一了 Lovable 暖中性配色，却刻意保留了两套布局，并禁止在聊天页使用永久侧栏。这种处理解决了局部样式冲突，但没有建立稳定的产品骨架，因此页面之间仍有明显割裂。

本次设计以 `/Users/wanyaozhong/Projects/tavily` 的本地复刻站点作为视觉结果参考，而不是代码、品牌或数据来源。参考中最重要的不是某个单独组件，而是固定侧栏、顶部状态区、受控内容宽度、暖色画布、低对比边界、克制产品蓝和紧凑排版共同形成的产品工作台感。

当前前端可以读取模型列表、Agent 模板、单个 Agent 的历史与事件，并在本地持久化最近会话引用；没有全局用量、额度或运行指标接口。因此概览页必须用现有事实构成，不得为了接近参考截图而制造数据。项目仍须保留明暗主题、本地化、键盘导航、减少动态效果，以及受保护组件目录不可直接修改的限制。

## 目标与非目标

**目标：**

- 为概览与隆中对建立统一、稳定、可响应的产品壳层。
- 复现 Tavily 参考的布局比例、视觉密度、颜色职责、表面层级、图标语言与交互节奏，而不是逐像素复制品牌页面。
- 把概览页变成由真实 API 与本地事实驱动的产品首页。
- 让隆中对在共享壳层内继续保持可读的对话宽度、固定 Composer、渐进式透明与可靠滚动。
- 用语义 token 与 Stratum 自有使用方完成迁移，并把最终规则更新为中文权威文档。

**非目标：**

- 复制 Tavily 的品牌名、Logo、业务文案、插图、额度展示或虚构指标。
- 在未确认授权的情况下复制 Suisse Intl 或参考项目中的其他字体文件。
- 新增后端聚合接口、全局会话索引、计费、用量统计或开发者编排功能。
- 修改路由路径、事件协议、Agent 运行语义或受保护可复用组件的内部实现。
- 增加环境式动画、玻璃拟态、霓虹、厚阴影或与产品状态无关的装饰特效。

## 决策

### 将 Tavily 作为结果参考，Lovable 作为底层基础

视觉优先级调整为：`PRODUCT.md` 的产品原则约束行为，Tavily 复刻站点定义最终工作台观感，Lovable 提供暖中性色与克制纵深的基础，Stratum 自有配色只用于标识及后续有目的特效。实现不会导入 Tavily 项目的组件或样式文件，而是把观察到的角色转换为 Stratum 语义 token。

浅色主题的核心角色为：

- 页面画布：`#fefcf5`
- 侧栏与轻强调表面：约 `#fdf8ec`
- 主文本：`#1f1e1e`
- 次级深色文本：约 `#2d2b2b`
- 品牌蓝、当前选择与焦点：`#2677ff`
- 填充主要操作：同色相的 `#1e65d9`，确保与白字达到 AA 对比度
- 普通边界：以主文本 8%–14% 透明度派生
- 运行就绪或成功：只使用可访问绿色语义色，不承担普通强调职责

暗色主题从相同暖色关系派生：暖炭黑为画布，略亮的中性表面建立层级，奶油色承担文本，产品蓝继续只用于操作与焦点。最终具体值在实现时通过两种主题的视觉与对比度验证确定，并归档到 `stratum-web/DESIGN.md`。

备选方案是完整保留上一轮 Lovable 炭黑主按钮和临时 Hero。未采用，因为用户已将 Tavily 复刻站点确定为最终风格，产品蓝与工作台骨架是其识别度和操作层级的重要组成部分。

### 使用一个共享产品壳层承载两个路由

桌面端采用三段结构：

```text
┌───────────────────┐  ┌────────────────────────────────────────────┐
│ 固定侧栏 285px    │  │ 顶部状态区 75px                           │
│ 距视口 16px       │  └────────────────────────────────────────────┘
│ 圆角约 14px       │  ┌────────────────────────────────────────────┐
│                   │  │ 主内容画布，最大宽度约 1024px              │
│ 概览              │  │                                            │
│ 隆中对            │  │ 路由内容                                   │
│ 最近会话（有则显）│  │                                            │
└───────────────────┘  └────────────────────────────────────────────┘
```

壳层负责品牌、一级导航、真实连接或配置状态、桌面侧栏、移动抽屉、内容偏移、安全间距和跳转后的焦点管理。路由只提供主内容，不各自重建导航。顶部状态区不得复制 Tavily 的额度信息；它只显示前端确实能够判断的连接、模型或配置就绪状态，无法判断时使用中性说明而不是成功状态。

桌面断点以下，固定侧栏退出布局，顶部状态区提供图标库菜单按钮打开模态抽屉。抽屉包含相同一级导航与最近会话，不允许出现横向滚动。桌面与移动使用同一数据源和语义组件，避免形成两套导航逻辑。

备选方案是只在概览页使用侧栏、隆中对进入无侧栏 Focus Mode。未采用，因为它会继续放大路由切换时的结构断裂；对话阅读宽度可以通过主内容内层控制，无须移除产品壳层。

### 把概览页设计为真实状态与入口的组合

概览页不模拟云控制台指标，而是组合已有事实：

1. 欢迎与主要操作：简洁说明产品用途，主要操作进入隆中对。
2. 就绪状态：模型端点、可用模型、Agent 模板的加载与可用情况。
3. 最近会话：读取本地持久化的会话引用，显示真实标题、更新时间和可恢复入口。
4. Agent 入口：展示后端返回的真实模板；不可用时显示清晰空状态或重试。

各区块使用 8–16px 圆角、低对比边框与极轻纵深。页面避免三张等宽通用卡片和卡片套卡片，优先使用不对称的主次布局：主要入口占更大视觉面积，状态与最近内容作为辅助区域。任何数据缺失都通过空状态表达，不以静态数字填补。

备选方案是继续保留 Hero，等待未来后端 Dashboard API。未采用，因为现有 API 和本地状态已经足以构成有用首页，并且 `PRODUCT.md` 已将当前 Hero 定义为临时占位。

### 隆中对复用壳层，但保持对话作为主任务

隆中对主区域继续使用文档滚动。对话正文内层约 800px，Composer 外宽最大约 896px；两者在壳层可用宽度中居中。固定 Composer 的水平定位基于主内容区域，而不是整个视口，避免被 285px 侧栏挤压或遮挡。底部安全空间由内容流提供，确保最后一条消息可以完整滚动到输入区上方。

桌面端最近会话直接出现在共享侧栏的次级区域；移动端放入同一导航抽屉。原有独立历史浮层不再作为桌面主入口，但其数据读取、会话恢复与空状态逻辑可以复用。侧栏不得展示工具事件、审批详情或开发者调试面板，避免把聊天变成运行控制台。

滚动到底部、下拉、关闭、发送、菜单等图形操作统一使用项目现有图标库，图标尺寸与约 1.8 的视觉线宽保持一致；不得使用 `↓`、`⌄`、`×` 等手写字符模拟图标。滚动到底部按钮只在用户离开真实底部时出现，点击后恢复自动跟随。

备选方案是把对话流扩展到主画布全宽。未采用，因为长行会降低中文与英文内容可读性，也会让聊天过度接近开发者日志界面。

### 保留字体家族，采用目标排版密度

在没有字体授权证据前，继续使用项目已有的 Noto Sans 与 Nunito Sans。通过语义字阶复现参考密度：工作台主标题最高约 44px，正文与对话不低于 16px，导航与紧凑控件通常为 14px，常规字重使用 400，强调使用 600；标题采用轻微负字距，控件保持可读的正常或微紧字距。中文与英文共用语义角色，不按页面单独设定字号。

如果未来确认 Suisse Intl 的授权，应以独立变更评估加载、多语言回退和性能；本次变更不把字体授权作为完成条件。

### 使用现有数据边界，不为视觉效果扩展后端

概览页通过现有前端服务层并行加载模型与模板，并读取本地最近会话。每个数据区块独立呈现加载、成功、空和失败状态，避免一个端点失败导致整个首页不可用。顶部状态只由同一批真实结果推导，不写入新的全局缓存或创建推测性 Dashboard adapter。

需要多个页面共享的数据，应复用当前查询或状态机制；若当前实现没有共享层，优先在壳层附近组合现有 hooks，而不是为了未来需求建立新的 manager、repository 或 snapshot 层。

### 通过 token、路由使用方和小型自有组件迁移

全局样式负责颜色、排版、圆角、边界、阴影与动效 token。共享壳层、概览区块和隆中对布局放在 Stratum 自有组件目录。`app/components/ui`、`app/components/react-bits` 与 `app/components/ai-elements` 的内部实现保持不变；需要调整时使用 props、className、CSS 变量、Stratum 包装层或限定在使用方的选择器。

动效只表达抽屉开合、导航切换、状态变化和滚动提示，目标为 150–250ms，优先动画 `transform` 与 `opacity`。`prefers-reduced-motion: reduce` 下直接呈现最终状态。不得为了接近参考加入背景漂浮、循环光效或滚动驱动装饰。

## 风险与权衡

- [固定侧栏压缩较小桌面视口的聊天空间] → 在约 1024px 附近切换为抽屉，并让 Composer 按主内容可用宽度计算，不使用固定视口坐标。
- [概览数据能力有限，成品可能显得内容偏少] → 使用清晰的不对称信息层级与有用空状态，不通过伪造指标增加密度；后续真实能力出现后再扩展。
- [浅色 Tavily 参考缺少完整暗色规范] → 从同一暖中性角色派生暗色 token，逐页面验证对比度、层级和截图，不引入蓝灰或紫灰表面。
- [固定侧栏中的最近会话可能让聊天用户感到拥挤] → 只显示有限条真实最近会话并提供截断，详细管理留给后续真实需求；无会话时使用极简空状态。
- [全局 token 调整影响受保护组件] → 先建立语义 token，再逐路由检查；只在 Stratum 使用方修正，不直接修改受保护组件。
- [参考项目存在硬编码颜色、原生字符图标和未确认字体授权] → 只提炼视觉规则，全部映射为语义 token和现有图标组件，不复制这些实现缺陷。
- [新变更与上一轮已完成但未归档的变更存在冲突] → 在实施时以本变更为准更新权威文档，保留仍有效的 Lovable 基础，并明确移除旧的 Hero 与无侧栏约束。
