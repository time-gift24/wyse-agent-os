Wyse 前端：原生事件优先

不引入 AI SDK；展示层不拥有协议与持久化事实。

模块边界
ChatWorkspace（现有布局）
消息流、输入框、取消、恢复、审批卡片、左侧本地最近会话
↕ 只接收 ViewModel 与 actions
useAgentConversation
React reducer + 恢复编排；把 API DTO / StreamEnvelope 投影为 UI 状态
wyse-api client
HTTP 请求、SSE 解码、API 错误
recent agents
localStorage，仅保存本机入口
↕ 原样保留 Wyse 契约
wyse-api
AgentView · HistoryPage · 原生 StreamEnvelope SSE · message / resume / cancel / approval

状态归属

服务端事实

  • 完整消息与 business_seq
  • Agent 状态、run / turn
  • 原生过程事件与 transport cursor
  • 工具审批的有效性

浏览器投影

  • 稳定消息的渲染顺序
  • 进行中的文本 / reasoning / 工具片段
  • 未完成审批的界面状态
  • 本机最近访问过的 Agent

后续只从 AI Elements 选择视觉组件源码；它们必须接收这里定义的 ViewModel，绝不接入 AI SDK 类型或 transport。
