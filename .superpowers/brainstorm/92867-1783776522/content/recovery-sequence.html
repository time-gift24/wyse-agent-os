恢复与实时合并

严格遵循已批准的 Web 恢复契约：SSE 先建立，历史后读取，消息按业务序号去重。

打开 / 刷新某个 Agent
1
先建立 SSE
首次：replay=all；刷新：按该 agent 保存的 after_cursor。立即把 retained + live 事件按到达顺序暂存。
2
读取 AgentView
取得状态与 last_seq，固定这次历史的上界。
3
分页读取 HistoryPage
持续请求 after_seq + 固定 through_seq=last_seq，构建完整稳定消息。
4
投影并排空 SSE 缓冲
以 (agent_id, business_seq) 去重稳定消息；过程事件只更新暂态 UI。之后同一条 SSE 直接进入 reducer。
稳定事实
  • HistoryPage / AgentEvent::Message → 按 business_seq 排序的对话消息
  • AgentView → status、run/turn、usage、last_seq
  • 已确认的文本、reasoning、tool calls 由完整消息承载
运行中投影
  • Llm TextDelta / ReasoningDelta → 当前草稿
  • ToolCall* → 工具过程详情
  • ToolApprovalRequested → 待处理审批卡
  • Finished / Failed / Cancelled → 状态与操作可用性

cursor 过期（410）

清除该 agent 的 localStorage cursor，关闭旧流，重新走完整恢复；不猜测或补造事件。

流内错误（stream_error）

保留已投影内容，标为连接错误并关闭；用户触发重连时使用最新保存的 cursor。
