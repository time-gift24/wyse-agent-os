All events consume seq today

Every delta calls the same reserve_event_seq().

No exact checkpoint event

Checkpoint stores a separate Vec<ChatMessage> BLOB, not these envelopes.

No recovery anchor

The stable ChatMessage and its JetStream delta sequence are not tied by an event carrying business seq.

Design question exposed by the current tree
We need stable, seq-bearing semantic events for user / assistant / tool history. The decision is whether to enrich existing semantic events or introduce one neutral message event — without adding a separate MessageCommitted lifecycle event.
