The anchor lookup maps business seq = L to JetStream's internal stream sequence; the client never mixes those counters.
Integration rule
A later stable event with seq is authoritative. It replaces the provisional content assembled from its preceding realtime deltas. No extra commit event is introduced.
Interval correction: we previously defined last_seq as the last committed seq (inclusive). If front_seq means “the last seq already held by the client,” the missing range is (front_seq, last_seq], not [front_seq, last_seq). The latter only works if one of the two fields is redefined as an exclusive cursor.
Retention boundary: if the JetStream anchor for last_seq has expired, stable history still recovers from the filesystem, but exact transient replay is impossible. The server must return a reset/gap signal and start from the earliest retained event or from live traffic.
